# Réalisation d'un dashboard et assurer une veille technique.

*Projet réalisé dans le cadre de la formation Datascience d'Openclassrooms*

**Contexte :**

L’entreprise souhaite mettre en œuvre un outil de “scoring crédit” pour calculer la probabilité qu’un client rembourse son crédit, puis classifie la demande en crédit accordé ou refusé. Elle souhaite donc développer un algorithme de classification en s’appuyant sur des sources de données variées (données comportementales, données provenant d'autres institutions financières, etc.)

**Objectifs :**

- réaliser un dashboard à destination de chargés de relation client afin de pouvoir mieux expliquer à leurs clients les décisions, et parfois les revoir si nécessaire.

- Déployer ce dashboard sur le cloud (heroku : https://projet-8-realiser-un-dashboard-32ccbf69e346.herokuapp.com)
